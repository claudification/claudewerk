## TL;DR
Two major reframes shipped this week and are now **live in main**: daemon is a *transport* of the claude backend (not a separate backend — URIs are `claude://`), and the chunked map-reduce **Recap v2.1** engine (merged at `ac4dfd98`, deployed). Profile no longer lives in project URIs — always canonicalize via `projectIdentityKey()`. Transcript performance is now driven by last-50 windowing + infinite scrollback, not full-DOM render.

## State
- **Daemon-as-transport is fully cut over.** `claude-pty`/`claude-headless`/`claude-daemon` replace the old backend enum; `defaultTransport` flipped to daemon. `backend:'daemon'` is gone (hard cutover) — any external caller sending it gets a zod enum error. Spawn dialog, conversation-item, launch-profile editor all key off transport.
- **Recap v2.1 is merged (`ac4dfd98`) and deployed** with broker rebuilt, web built, agent-host packages rebuilt. ## Decisions
- **Daemon is a transport, not a backend** — URIs stay `claude://`; this is the same lesson as profile-url-strip. It forces headless-vs-daemon feature gaps into the open and cleans the data model. (`78b962fe-788`, `5a7e6074`)
- **Hard cutover, no back-compat** on both daemon and profile-strip — we control all first-party clients; a zod enum error is enough for externals. (`8ee9accb-a4e`, `4a0318fc-d7a`)
- **Reaper's primary signal is `liveSockets===0`, not `lastActivity`** — reconnects are legitimate and were corrupting idle tracking. (`c27e0ef1`)
- **Recap renders React components, not LLM-emitted HTML** — type-safe, themed, no XSS; one tree drives in-app + public share. Chunking uses cheap Sonnet for map / Opus for synthesis / code-only dedup; retrospect is an opt-in Opus-only layer. (`a4b434c8`, `220627c6`)
- **Stay on Biome; oxc not yet** — oxfmt 0.52 beta, oxlint missing exhaustive-deps/jsx-a11y, and a whole-tree reformat would conflict-storm the ~15 active worktrees. (`87f16e52`)
- **Profile NAME only crosses the wire** — sentinel knows the profile but never propagates env/keys, preserving the Profile-Env boundary. (`0e7dd367`)

## Gotchas
- **Zustand object-returning selectors cause infinite re-renders** (React #185) — Object.is fails on fresh objects. Use per-field selectors or `useShallow`. This recurred multiple times; documented in `.claude/topics/gotchas-frontend.md`.
- **Biome `--write` reformats the whole tree, not just changed files** — `bun run lint` pollutes commits and creates rebase conflict surface. Stage by explicit path; revert collateral before committing.
- **`projectSettings[conv.project]` raw lookup misses legacy/leaked keys** — always go through `normalizeProjectUri`/`projectIdentityKey`.
- **FTS5 treats unquoted hyphen as NOT** — `war-council` becomes `war NOT council`; quote hyphenated terms (`sanitizeFtsQuery` handles this).
- **`git cherry` lies on squash/rebase merges** — use `git merge-tree --write-tree` for reliable integration status.
- **Broker deploy recreates the container and drops all live WebSocket connections** — `bun run deploy` is irreversible; requires explicit authorization. Web-only changes deploy via bind-mounted `web/dist` with no restart.
- **Agent-host binary lags broker by days** — already-running processes keep the old bundle after deploy; only new spawns get the new binary. Tests of new MCP tools must spawn fresh.
- **Worktree Edit/Write paths must be absolute under the worktree dir** — shell `cd` does not redirect tool paths; you'll edit MAIN's tree by accident.
- **`exit_conversation` has no target param** — calling it terminates the *calling* agent; use `control_conversation {action: quit}` for peers.
- **SW Cache-API responses make `PerformanceResourceTiming` opaque** (all size fields zeroed); probe `caches.match()` for real byte size.
