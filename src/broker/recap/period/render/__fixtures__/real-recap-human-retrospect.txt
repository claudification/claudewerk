discoveries:

  - 216KB WebSocket fanout messages cause 1.8s paint stalls
  - Conversation-switch stalls are browser layout/paint (~1.3s), not fetch or JS

  - FTS5 tokenizer treats hyphens as a NOT operator unless quoted
  - Daemon mirrors have no socket by design, so liveSockets===0 is normal, not phantom
  - daemon:// vs claude:// scheme mismatch splits projectIdentityKey buckets and breaks grouping
  - React-doctor found 30 fixable issues (24 only-export-components, 6 effect cleanup)

  - OpenRouter real billed cost is only available via usage.include
  - git cherry lies on squash merges; merge-base + merge-tree is accurate
  - Thinking token count exists in the protocol but is not persisted to the cost rollup
  - Transport buffering/lag produces false timeout alarms (output arrives batched)
  - recap_create schema enum is 'last_7', not 'last_7_days'

  - URI format change is breaking and requires migration

  - Broker container recreation required for transport deploy
  - New MCP tools require a fresh conversation spawn to access (old agent bundle cached)

## Decisions
- **Profile as launch metadata, not URI identity** — keeping profile in the URI caused identity fragmentation and a permission-matrix explosion ([4a0318fc](/sessions/conv_4a0318fc)).
- **Hard cutover for the daemon backend** — daemon is a transport; removing `backend:'daemon'` outright avoids carrying a wrong abstraction forward ([8ee9accb](/sessions/conv_8ee9accb)).
- **MCP tools live on the broker; agent hosts are dumb pipes** — broker owns all schemas/handlers so heterogeneous clients evolve without touching the wire ([36c46c4e](/sessions/conv_36c46c4e), `b1c020be`, `f96a1231`).
- **Skip daemon conversations in the phantom reaper** — mirrors are socket-less by design, so `liveSockets===0` is normal, not phantom ([02ce729d](/sessions/conv_02ce729d), `78d735cb`).
- **Recover orphans from the `turns` table** — `project_uri` there is authoritative for 445 of 648 orphans, since the broker can't derive URIs from transcript JSON ([beba12c1](/sessions/conv_beba12c1)).
- **Stay on Biome** — oxfmt is too early (0.x beta) and oxlint has rule gaps ([87f16e52](/sessions/conv_87f16e52)).
- **Scribe/Worker split** — an expensive Worker does the work while a cheap Scribe handles coordination, addressing cost ([1a9c2e83](/sessions/conv_1a9c2e83)).
- *Inferred:* RecapId-keyed artifact bundles for resumability ([389972df](/sessions/conv_389972df)); retrospect as an optional recap mode ([1f0bd0ba](/sessions/conv_1f0bd0ba)); defer thinking-token analytics to v2.2 ([a135e015](/sessions/conv_a135e015)).

## Dead ends
- **Profile in URI** — abandoned for identity fragmentation and permission explosion ([4a0318fc](/sessions/conv_4a0318fc)).
- **Daemon as a separate backend** — wrong abstraction level, reframed as transport ([78becbbf](/sessions/conv_78becbbf)).
- **Modifying the agent-host MCP HTTP client** — the agent host only receives MCP calls, it never makes outbound ones ([3352290f](/sessions/conv_3352290f)).
- **Tmux send-keys for the PTY initial prompt** — rejected ([fd308e75](/sessions/conv_fd308e75)).
- **CWD scanning from transcript content** — broker can't derive project URIs from transcript JSON; cwd only exists as agent metadata ([beba12c1](/sessions/conv_beba12c1)).
- **Per-entry virtualization inside groups** — high regression risk; progressive loading was simpler ([576c695f](/sessions/conv_576c695f)).
- **`git cherry` for integration detection** — lies on squash merges; use `merge-base` + `merge-tree` ([87f16e52](/sessions/conv_87f16e52)).
- **TTL-based SOTU freshness** — decay by integration status is more accurate than time ([1f0bd0ba](/sessions/conv_1f0bd0ba)).

## Gotchas
- **Zustand object selectors** fail `Object.is` equality and loop infinitely ([b5e47b7a](/sessions/conv_b5e47b7a)).
- **FTS5 treats `-` as NOT** unless the term is quoted ([cc78179a](/sessions/conv_cc78179a)).
- **Daemon mirrors have no socket by design** — don't treat `liveSockets===0` as phantom ([02ce729d](/sessions/conv_02ce729d)).
- **`daemon://` vs `claude://`** split `projectIdentityKey` buckets and break grouping ([5a7e6074](/sessions/conv_5a7e6074)).
- **Group-level virtualization renders the whole transcript** — 500 entries collapse to 8 groups, all 8 lay out at once ([576c695f](/sessions/conv_576c695f)).
- **commit→paint timing includes tab-suspension artifacts**, producing false 3s+ stalls ([3860a533](/sessions/conv_3860a533)).
- **Tables linked by convention not constraint** — no FK between `conversations` and `transcript_entries` allows orphaning ([beba12c1](/sessions/conv_beba12c1)).
- **New MCP tool schemas cache at client init** — running conversations need a reconnect / fresh spawn to see new params ([36c46c4e](/sessions/conv_36c46c4e), [a135e015](/sessions/conv_a135e015)).
- **Worktree edits use absolute paths** (shell `cd` doesn't redirect them); branch collisions crash spawn hooks; deploy requires main-repo coordination ([bc2fbde9](/sessions/conv_bc2fbde9), [e3cf0e71](/sessions/conv_e3cf0e71), [1cb88e4f](/sessions/conv_1cb88e4f)).
- **Biome reformats the whole tree on lint**, producing noise commits ([389972df](/sessions/conv_389972df)).
- **OpenRouter `usage.include` is required** for real billed costs vs estimates; **agent-host role is blocked** from many MCP operations ([9d206f82](/sessions/conv_9d206f82), [23281bf0](/sessions/conv_23281bf0)).

## Retrospective

### What went well
The big architectural bet of this period — collapsing the daemon into a transport and pulling profile out of URI identity — actually shipped end-to-end with a coordinated deploy and recovery docs, rather than stalling as a design ([8ee9accb](/sessions/conv_8ee9accb), [eb91fd37](/sessions/conv_eb91fd37)). The flapping storm got a genuine root cause (socket-less mirrors + reconnect re-stamping) instead of a band-aid ([02ce729d](/sessions/conv_02ce729d), [c27e0ef1](/sessions/conv_c27e0ef1)). Performance work was disciplined: the team first proved the stalls were browser paint and tab-suspension artifacts before changing code ([3860a533](/sessions/conv_3860a533), [576c695f](/sessions/conv_576c695f)). And Recap v2.1 landed four pillars plus real cost tracking in a single coherent push ([a135e015](/sessions/conv_a135e015), [9d206f82](/sessions/conv_9d206f82)).

### What went badly
The React #185 selector crash reached the header and broke every page load — a recurring Zustand footgun that should have been caught by tooling ([b5e47b7a](/sessions/conv_b5e47b7a)). Spawn/worktree handling was fragile all week: branch collisions, children landing in the wrong directory, and an agent that killed itself with `exit_conversation` ([e3cf0e71](/sessions/conv_e3cf0e71), [0b78b579](/sessions/conv_0b78b579)). Multi-agent work on a shared repo created coordination overhead, worktree collisions, and tree-wide biome reformat noise ([5ae005b8](/sessions/conv_5ae005b8), [389972df](/sessions/conv_389972df)). Recap testing also burned cycles on avoidable mismatches — the `last_7_days` vs `last_7` enum and agent-host role blocks — and a failed $10+ recap recorded $0 cost ([1aadb461](/sessions/conv_1aadb461), [23281bf0](/sessions/conv_23281bf0), [9d206f82](/sessions/conv_9d206f82)).

### Recommendations
1. **Add a lint/CI rule banning object-returning Zustand selectors** — the #185 crash is recurrent and a header outage is high-cost; require shallow-equality or per-field selectors ([b5e47b7a](/sessions/conv_b5e47b7a)).
2. **Make WorktreeCreate spawn hooks idempotent** — detect and reuse existing branches/worktrees instead of crashing, which repeatedly broke spawns ([e3cf0e71](/sessions/conv_e3cf0e71), [0b78b579](/sessions/conv_0b78b579)).
3. **Document the fresh-spawn requirement for new MCP tools in CLAUDE.md** — tool schemas cache at client init, and this was rediscovered mid-test ([36c46c4e](/sessions/conv_36c46c4e), [a135e015](/sessions/conv_a135e015)).
4. **Record cost on failed recap runs** — close the pillar-E gap so $10+ failures don't report $0 ([9d206f82](/sessions/conv_9d206f82)).
5. **Pin `recap_create` enum (`last_7`) and allowed roles in docs** — stop wasted calls from wrong period labels and role blocks ([1aadb461](/sessions/conv_1aadb461), [23281bf0](/sessions/conv_23281bf0)).
6. **Adopt a per-agent pathspec / no-mass-format convention** — to cut biome reformat noise and worktree edit conflicts in multi-agent builds ([5ae005b8](/sessions/conv_5ae005b8), [389972df](/sessions/conv_389972df), [87f16e52](/sessions/conv_87f16e52)).
